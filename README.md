# 📊 Service Performance KPI Analysis

## Overview
This project focuses on building a data pipeline and analytical foundation to track customer service performance and commercial targets.

The goal is to transform raw operational data into structured datasets that support performance evaluation at the agent level.

---

## Business Problem
Customer service agents are evaluated based on multiple performance metrics, including service volume, customer data accuracy, and commercial outcomes.

However, raw data is spread across different systems, making it difficult to:
- Track individual performance
- Ensure data consistency
- Monitor key KPIs over time

---

## Solution
This project creates a structured data layer by integrating and transforming raw service data.

The pipeline:
- Extracts customer service records
- Joins organizational and reference data
- Applies business rules to filter and clean records
- Outputs a curated table for analysis and dashboarding

---

## Key Metrics
- Number of customer interactions
- Data registration effectiveness
- Digital billing adoption
- Negotiation performance indicators (e.g., average down payment rate)

---

## Technologies Used
- PySpark
- Spark SQL
- Data Lake architecture
- Data modeling for analytics

---

## Data Pipeline

1. Extract raw service data  
2. Join with reference tables (city, region)  
3. Apply filters and business rules  
4. Save curated dataset for BI consumption  

---

## Disclaimer
All data structures, names, and fields have been anonymized to preserve confidentiality.  
This project is intended for portfolio purposes only.
